

## 프로젝트 구조

### 1. 하드웨어

- 라즈베리파이4 (로봇의 메인 보드)
- USB/Web 카메라
- 라이다 센서 (SLAM + 주행용) + waffle
- 모터/ROS2 기반 제어보드

### 2. 소프트웨어 역할 분리

- ROS2(Humble)
    - SLAM 노드 (라이다 데이터로 지도 및 위치 추정)
    - Nav2 같은 경로 계획 노드
- OpenCV (라즈베리파이 카메라 입력 처리)
    - ArUco 마커 인식 (카메라 프레임 → 마커 ID 검출)
    - "특정 마커 ID 발견 시 → 도착 이벤트 발행"

### 3. 연동 방식

- OpenCV 마커 검출 → 결과를 ROS2 토픽으로 퍼블리시
    `(예: /aruco_marker/detected_id)`

- Nav2 혹은 제어 노드가 이 토픽을 구독해서:
    - 마커 확인 → "목적지 도착"으로 판단
    - 로봇 멈춤, 알림음 출력, 트레이 열기 등 동작 실행
